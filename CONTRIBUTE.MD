# Contributing Angel & Co TLDR (Too Long Didn't Read) AI New Scraper and Summarizer

## The Workflow

1. Branching - Remember when working on stories do not push directly to main. Create a branch named as the specific story you are working on.
Ex: 'git checkout -b ST-##"

2. Commit Messages - Keep your commit messages consistent with the work you are committing, be descriptive on what work you are committing and what was done in your commit.
Ex: 'feat: Added <insert feature>' or 'fix: fixed <insert error>'

3. Pushing - Push to github frequently
Ex: 'git push -u origin main'

4. Pulls - Rememeber to frequently pull or fetch to keep your code up-to-date with any new changes before pushing or merging.
Ex: 'git pull <repo>' or 'git fetch <repo>'

5. Merging - When your story is done and your ready to merge your branch back to main remember to checkout main, merge, then push. Also remember to commit and push in the backend before merging.
Ex: 'git checkout main" then 'git merge <Branch Name>' finally 'git push -u origin main'

## Coding Standards

Linter - Ensure your code passes the project linter before submitting

Structure - Keep your code neat and readible, make sure everything is organized in a way for others to easily read or find certain parts of your code.

Documentation/Comments - Write comments under any new function to explain its functionality and how it is used. Explained its parameters and return values as well.
